TROUBLESHOOTING 좋아요 카운트 동시성 경합
문제
가장 자주 호출되는 좋아요에서 동시 요청 시 카운트가 어긋나는 경합(Lost Update)이 발생.
원인·가설
DB 카운터만으로 처리하면 SADD 확인 없이 INCR가 겹쳐 중복 좋아요·카운트 오차가 생긴다고 판단.
해결
Redis Set으로 사용자 중복을 막고, Lua 스크립트로 검사와 증감을 원자적으로 묶어 경합 구간 제거. 캐시 장애 시 DB 폴백까지 설계.
결과
동시 요청에서 카운트 정합성 보장. 사람이 놓치기 쉬운 동시성 결함을 구조로 차단.
TROUBLESHOOTING 반복 결함을 사람이 매번 못 잡음
문제
협업 중 트랜잭션 누락·soft delete 위반·Redis Lost Update 같은 반복 결함을 사람이 PR마다 잡기 어려움.
가설
프로젝트 규칙을 AI 리뷰어에게 명시적으로 규정하면 사람이 놓치는 패턴을 자동으로 걸러낼 수 있다.
해결
claude-code-action을 PR 열림에 트리거. 심각도 4단계·변경 라인만 리뷰·지적 5개 초과 시 축약 등 리뷰 규칙을 직접 설계하고 gh api로 코멘트 자동 게시.
결과
중복 좋아요(SADD 미확인 INCR)·IDOR 같은 결함을 머지 전에 차단. 리뷰 부담과 누락 감소.
